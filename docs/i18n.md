# Internationalization (i18n) Guide

ThreeDocsy supports multiple languages via a lightweight custom i18n system built with React Context.

## Supported Languages

| Code | Language |
|------|----------|
| `pl` | 🇵🇱 Polish (default) |
| `en` | 🇬🇧 English |

## Architecture

```
src/i18n/
├── index.ts              # Core translate() function, locale storage
├── pl.ts                 # Polish translations
├── en.ts                 # English translations
├── LanguageContext.tsx   # React context + LanguageProvider
└── LanguageDropdown.tsx  # UI dropdown component
```

## Usage in Components

### 1. Import and use the hook

```tsx
import { useLanguage } from '../../i18n/LanguageContext';

const MyComponent = () => {
  const { t } = useLanguage();
  return <h1>{t('dashboard.title')}</h1>;
};
```

### 2. With interpolation

```tsx
t('footer.copyright', { year: 2024 })
// "© 2024 ThreeDocsy"
```

### 3. Add the language dropdown

```tsx
import { LanguageDropdown } from '../../i18n/LanguageDropdown';

// In your component's JSX:
<LanguageDropdown />
```

## Adding New Keys

1. Add the key to `src/i18n/pl.ts` with the Polish value.
2. Add the same key to `src/i18n/en.ts` with the English value.
3. Use `t('your.new.key')` in your component.

The translation object is deeply nested. Use dot-notation keys to access nested values:

```ts
// pl.ts
export const pl = {
  myFeature: {
    title: 'Mój tytuł',
    description: 'Opis {count} elementów',
  }
} as const;

// Usage
t('myFeature.title')                    // "Mój tytuł"
t('myFeature.description', { count: 5 }) // "Opis 5 elementów"
```

## Adding a New Language

1. Create `src/i18n/XX.ts` (where XX is the language code) with translations matching the structure of `pl.ts`.
2. Add the new locale to `src/i18n/index.ts`:
   ```ts
   import { xx } from './xx';
   export type Locale = 'pl' | 'en' | 'xx';
   const translations: Record<Locale, typeof pl> = { pl, en, xx };
   ```
3. Update `getStoredLocale()` to accept the new code.
4. Add the option in `LanguageDropdown.tsx`.

## Language Persistence

The selected language is stored in `localStorage` under the key `language`. It persists across browser sessions.
